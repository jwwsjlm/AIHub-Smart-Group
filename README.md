# AIHub 智能分组

一个用于 AIHub 的 Tampermonkey 油猴脚本。它会读取供应商监控数据，按价格、首 Token 速度和可用率推荐分组，并可把已有 API 密钥切换到推荐分组。

脚本不会显示或保存完整 API Key。面板顶部会显示当前账户余额，并在每次检测刷新时重新读取。

## 支持页面

| 页面 | 功能 |
| --- | --- |
| `https://aihub.top/*` | 登录后在所有页面显示智能分组面板或右下角 `AI` 按钮 |
| `https://aihub.top/providers` | 读取供应商监控并推荐分组 |
| `https://aihub.top/keys` | 选择密钥、手动切换或开启自动切换 |
| `https://aihub.top/usage` | 额外在分组名称旁显示当前倍率 |

脚本只在 AIHub 登录状态下启用，不会在其他网站运行。通过侧边栏切换页面时不需要手动刷新。

## 安装

### 方法一：Greasy Fork 安装（推荐）

1. 在浏览器中安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展。
2. 打开 [AIHub Smart Group 的 Greasy Fork 页面](https://greasyfork.org/zh-CN/scripts/587603-aihub-smart-group)。
3. 点击绿色的“安装此脚本”按钮。
4. 浏览器会跳转到 Tampermonkey 安装页面，再点击一次“安装”。
5. 登录 [AIHub](https://aihub.top/)，打开“供应商大厅”“API 密钥”或“使用记录”页面即可使用。

以后发布新版本时，Tampermonkey 会定期检查更新。也可以打开 Tampermonkey 管理面板，在“已安装脚本”中手动检查更新。

### 方法二：GitHub 直接安装

1. 先安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展。
2. 打开 [脚本直装地址](https://raw.githubusercontent.com/jwwsjlm/AIHub-Smart-Group/main/aihub-smart-group.user.js)。
3. Tampermonkey 会自动识别脚本并显示安装页面，点击“安装”。
4. 登录 [AIHub](https://aihub.top/) 并刷新目标页面。

如果打开直装地址后只看到代码，没有出现安装页面，请确认 Tampermonkey 已启用，然后重新打开该地址。

### 方法三：手动安装

1. 点击 Tampermonkey 图标，选择“添加新脚本”。
2. 打开仓库中的 [aihub-smart-group.user.js](./aihub-smart-group.user.js)，复制全部代码。
3. 删除编辑器内原有示例代码，粘贴脚本代码并按 `Ctrl+S` 保存。
4. 确认脚本状态为“已启用”，然后刷新 AIHub 页面。

### 安装后没有显示

- 确认 Tampermonkey 和“AIHub 智能分组”脚本都已启用。
- 确认当前域名是 `https://aihub.top/`。
- 更新或重新安装脚本后，刷新 AIHub 页面。
- 登录后，智能推荐面板或右下角 `AI` 按钮会在所有 AIHub 页面显示；“使用记录”页面还会在分组名称旁显示绿色倍率。

## 第一次使用

1. 打开 `https://aihub.top/keys`。
2. 选择价格、平衡或速度模式。
3. 在“目标密钥”中选择要管理的密钥。
4. 点击“检测”。
5. 等推荐连续通过检测后，点击“切换到推荐分组”并确认。

目标密钥选项会依次显示密钥名、当前分组、倍率和最新首 Token，例如：`main · A001-K12 · ×0.05 · 首 Token 1141 ms`。选中后，下方详情区会完整显示这四项信息；如果当前分组没有对应监控数据，会明确显示“暂无数据”。脚本不会显示或保存完整 API Key。

自动切换默认关闭。打开前会弹出确认；打开后，只有推荐稳定、监控数据未过期、目标密钥不在推荐分组且冷却时间结束时才会切换。

## 三种模式

| 模式 | 选择规则 | 适合场景 |
| --- | --- | --- |
| 价格 | 选择价格倍率最低的可用分组 | 预算优先 |
| 平衡 | 在设置的最高倍率以内，选择首 Token 最快的分组 | 兼顾价格和速度 |
| 速度 | 选择首 Token 最快的可靠分组 | 响应速度优先 |

### 平衡模式如何工作

平衡模式使用一个直接的倍率上限：

1. 先按当前可靠性设置过滤分组，例如当前状态、最近 10 分钟可用率和监控警告。
2. 排除倍率高于“允许切换的最高倍率”的分组。
3. 在剩余分组中选择首 Token 最快的一个。

默认最高倍率为 `0.1`。例如设置为 `0.1`：

- `×0.04`、`×0.08` 和 `×0.1` 可以参与比较。
- `×0.11` 和 `×0.2` 会被直接忽略。
- 如果没有分组满足上限和可靠性条件，平衡模式不会推荐或自动切换。

设置方法：

1. 打开右下角智能分组面板。
2. 将“模式”选择为“平衡”。
3. 展开“设置”。
4. 在“允许切换的最高倍率”中填写数字，例如 `0.1`。
5. 点击“保存设置”，等待重新检测。

输入倍率时，设置下方会实时显示符合上限的分组数量，例如：`只考虑倍率 ≤ ×0.1 · 6 个分组可选 · 将选首 Token 最快`。输入阶段仅用于预览，出现“未保存”时不会改变推荐或自动切换目标；点击“保存设置”后才会正式生效。

保存后，平衡模式推荐卡片会显示：`倍率上限 ×0.1 · 范围内首 Token 最快`。

最高倍率是固定的绝对上限，不会随当前最低倍率变化，也不会绕过可用性和警告过滤。它会同时限制推荐卡片、手动切换和自动切换目标。

## 设置与保存

桌面端面板使用左右双栏：左侧用于检测、推荐和密钥切换，右侧通过“设置 / 日志”标签切换内容。两栏分别滚动，面板不会因为打开设置或查看大量日志而超出屏幕顶部。脚本会记住最后打开的“设置”或“日志”标签。窄屏设备会自动恢复为单列，并在面板内部滚动。

设置区域按“可靠性筛选”“检测与切换”“平衡策略”分区，可以调整：

- 最近 10 分钟最低可用率（可自行修改，`0.1` 表示 `10%`）
- 可用性判断方式：按百分比、按成功监控点数或按连续成功点数
- 连续通过次数
- 检测间隔
- 自动切换冷却时间
- 平衡模式允许切换的最高倍率
- 排除分组关键词
- 是否排除监控警告
- 是否开启自动切换

点击“保存设置”后，配置会写入当前浏览器的 Tampermonkey 存储。模式、目标密钥和最近一次切换记录也会保存。

面板顶部会显示当前账户余额，并在每次检测时动态刷新；余额不会写入日志或配置。

“排除分组关键词”使用 `|` 分隔多个关键词，例如 `free|unstable`。匹配忽略大小写，只要分组名称包含其中任一关键词，该分组就不会参与价格、平衡或速度模式的推荐和切换。输入时会实时显示命中的分组数量和部分名称，保存后才正式生效。

可用性判断默认使用最近 10 分钟可用率。由于 AIHub 监控通常每 5 分钟产生一个点，也可以切换为“成功监控点数”或“连续成功点数”：前者表示窗口内累计成功多少个点，后者表示最近的连续成功点数。例如设置为 `2`，前者允许前后两个点成功，后者要求最近两个点都连续成功。

切换冷却支持小数，例如 `0.1` 分钟表示 6 秒。设置下方会显示分钟换算结果；冷却期间还会实时显示剩余时间。

## 默认推荐规则

- 当前监测状态必须可用。
- 最近 10 分钟必须有有效探测样本，可用率至少 `10%`。
- 10 分钟可用率阈值可在设置中自行修改。
- 默认排除带监控警告的分组。
- 推荐连续通过默认 `2` 次检测后才视为稳定。
- 自动切换默认关闭，默认冷却时间为 `10` 分钟。
- AIHub 监控样本通常约每 5 分钟更新一次。脚本以序列中的最新实际样本为准，固定超过 `600` 秒（10 分钟）未更新才视为过期；这个保护值不开放手动调整。过期时推荐卡片会变红并写入一条错误日志，恢复后会写入一条恢复日志；推荐仍会显示，但手动和自动切换都会暂停。

脚本通过 AIHub 的分钟级监控序列计算最近 10 分钟可用率。价格模式按倍率升序排列；倍率相同时依次比较最近 10 分钟可用率和首 Token 延迟。6h、24h 等长期历史不会阻止近期可用的分组进入推荐。推荐卡片会显示监控数据年龄、实际参与比较数量，以及关键词、不可用、可用率不足和监控警告分别排除了多少分组。

供应商监控数据仍按设置的检测间隔刷新。密钥列表会在首次加载、手动检测和缓存超过 5 分钟时重新读取；密钥接口短暂失败时会保留已经读取的列表，避免目标密钥突然消失。

## 使用日志

面板中的“使用日志”会记录：

- 首次检测、手动检测及推荐结果变化
- 密钥读取或监控检测失败，以及恢复状态
- 模式或设置变化
- 自动切换开关变化
- 手动或自动切换结果
- 自动切换跳过原因（状态变化时记录一次）
- 请求失败和切换失败

最多保存最近 100 条，可以随时清空。日志只保存时间、动作、分组名和错误摘要，不保存完整 API Key、登录令牌或请求正文。

## 切换与安全

- 手动切换前会弹出确认框。
- 自动切换只修改已选密钥的分组，不创建新密钥。
- 登录令牌只从当前 AIHub 页面临时读取，用于当前请求。
- 完整 API Key 和登录令牌不会写入日志或脚本存储。

## 常见问题

### 面板没有出现

确认已登录 `aihub.top`，并在 Tampermonkey 中启用了脚本。更新脚本后刷新页面；如果旧面板仍在，检查是否重复安装了旧脚本。

### 显示“未找到页面登录令牌”

请在保存登录状态的同一个 Chrome 配置中重新登录 AIHub，然后刷新 `/keys` 页面。

### 显示“密钥接口返回 401”

当前登录已经失效。重新登录后点击“检测”。

### 没有推荐分组

检查“最近 10 分钟最低可用率”和“排除监控警告”设置。可用率输入范围是 `0–1`，例如 `0.1` 表示 `10%`、`0.9` 表示 `90%`。最近 10 分钟没有有效探测样本的分组不会参与推荐。也可以展开使用日志查看检测结果或错误摘要。

### 切换按钮是灰色

常见原因包括：尚未选择目标密钥、推荐没有连续通过规定次数、监控数据已经过期、当前密钥已经在推荐分组，或者检测仍在进行。把鼠标移到按钮上可以查看具体原因。
