# AIHub + 帅 API 智能分组

一个 Tampermonkey 油猴脚本，帮助你在 AIHub 和帅 API 里选择合适的分组。

它会读取当前账号能看到的性能数据，按价格、首字速度和可靠性给出推荐；需要切换时，仍由你确认后执行。脚本不会显示或保存完整 API Key。

## 支持页面

| 网站 | 页面 | 功能 |
| --- | --- | --- |
| AIHub | `https://aihub.top/providers` | 读取供应商监控、按模式排序分组 |
| AIHub | `https://aihub.top/keys` | 选择密钥并切换分组，支持自动切换 |
| 帅 API | `https://api.shuaiapi.com/performance` | 按模型查看推荐、选择密钥并切换分组 |

脚本只在上面三个页面运行，不会向其他网站注入面板或发送请求。

## 安装

### Greasy Fork 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/)。
2. 打开 Greasy Fork 上的脚本页面并点击安装。
3. 登录 AIHub 或帅 API，然后打开对应的支持页面。

### GitHub 手动安装

1. 安装 Tampermonkey。
2. 打开脚本文件：[aihub-smart-group.user.js](./aihub-smart-group.user.js)。
3. 复制全部内容，在 Tampermonkey 中新建脚本，粘贴并保存。
4. 登录网站后刷新页面。

只需要安装这一个脚本。它会根据当前网址自动启用 AIHub 或帅 API 功能。

## 第一次使用

### AIHub

1. 打开 `aihub.top/keys`。
2. 在“模式”中选择价格、平衡或速度。
3. 在“目标密钥”中选择要管理的密钥。
4. 点击“检测”。
5. 等推荐稳定后，点击“切换到推荐分组”并确认。

AIHub 的自动切换默认关闭。打开前会弹出确认；打开后，只有推荐连续通过检测、目标密钥不在当前分组且冷却时间结束时才会切换。

### 帅 API

1. 打开 `api.shuaiapi.com/performance`。
2. 选择模型或模型类型，以及 24h 或 7d 时间窗。
3. 选择模式，点击推荐卡片可以把对应分组写入“目标分组”。
4. 选择目标密钥，点击切换按钮并确认。

帅 API 默认只推荐最近时间桶有请求、成功率至少 99% 且未被标记为不稳定的分组。

## 三种模式

| 模式 | 选择规则 | 适合场景 |
| --- | --- | --- |
| 价格 | 选择倍率最低的可用分组 | 预算优先 |
| 平衡 | 在最低倍率上浮指定百分比的范围内，选择首字最快的分组 | 价格和速度都要兼顾 |
| 速度 | 选择首 Token/TTFT 最快的可靠分组 | 响应速度优先 |

平衡模式的价格范围默认是 `20%`，可以在“设置”中修改。范围越大，速度模式可能选择更高倍率的分组。

AIHub 的模式会同时影响推荐卡片、手动切换和自动切换目标。帅 API 的模式会影响“目标分组”的自动选择；实际切换仍需要点击确认。

## 设置与保存

AIHub 设置包括：

- 6h 和 24h 最低可用率
- 连续通过次数
- 检测间隔
- 切换冷却时间
- 平衡价格范围
- 是否排除监控警告
- 是否开启自动切换

帅 API 设置包括：

- 24h / 7d 时间窗
- 模型或模型类型
- 最低成功率
- 平衡价格范围
- 是否要求最近有数据
- 是否排除不稳定分组
- 是否让模式自动选择目标分组
- 刷新间隔

点击“保存设置”后，设置会保存到当前浏览器的 Tampermonkey 存储中。模式、选择的密钥和目标分组也会保存。换浏览器配置后需要重新设置。

## 日志

两个面板都有“使用日志”。日志会记录：

- 检测完成和推荐结果
- 模式或设置变化
- 目标分组选择
- 手动或自动切换成功
- 请求失败和切换失败

最多保存最近 100 条日志，可以在面板中展开查看或清空。日志只保存时间、动作、分组名和错误摘要，不保存完整 API Key、登录令牌或请求正文。

## 推荐规则

### AIHub 默认规则

- 当前监测状态必须可用。
- 6h 可用率至少 `95%`。
- 24h 可用率至少 `90%`。
- 默认排除带监控警告的分组。
- 价格模式按倍率升序排列；同倍率时比较 6h 可用率、24h 可用率和首 Token 延迟。
- 推荐连续通过默认 `2` 次检测后才视为稳定。
- 自动切换默认关闭，默认冷却时间为 `10` 分钟。

### 帅 API 默认规则

- 默认使用 24h 数据。
- 默认最低成功率为 `99%`。
- 默认要求性能时间序列的最新时间桶有请求。
- 默认排除页面标记为不稳定、警告或故障的分组。
- 如果历史有流量但当前时间桶没有请求，该分组不会被推荐。

## 切换与安全

- 手动切换前会弹出确认框。
- AIHub 自动切换只修改已选密钥的分组，不创建新密钥。
- 帅 API 切换调用站点自己的分组更新接口。
- 脚本不会输出完整 API Key。
- AIHub 登录令牌只从当前页面临时读取，用于当前请求，不写入脚本存储。

## 常见问题

### 面板没有出现

确认当前网址是支持页面，并在 Tampermonkey 中启用了脚本。更新脚本后刷新页面；如果旧面板仍在，先禁用重复安装的旧脚本。

### AIHub 显示“未找到页面登录令牌”

请在保存登录状态的同一个 Chrome 配置中重新登录 AIHub，再刷新 `aihub.top/keys`。

### AIHub 显示“密钥接口返回 401”

当前登录已失效。重新登录后点击“检测”。

### 帅 API 显示“未读取到密钥”

确认已登录帅 API，并在 `api.shuaiapi.com/performance` 页面刷新。若性能数据能读取但密钥不能读取，查看“使用日志”中的错误摘要。

### 推荐分组为空

检查模型筛选、最低成功率、最近数据和排除不稳定设置。可以先选择“全部模型”、使用 24h，并适当降低最低成功率。
