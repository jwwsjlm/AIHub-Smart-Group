# 帅API智能分组设计

## 目标

在现有 `aihub-smart-group.user.js` 中增加帅API性能页的只读分组推荐能力，用户只安装一个 Tampermonkey 脚本即可使用 AIHub 和帅API功能。

## 数据与边界

- 仅在 `https://api.shuaiapi.com/performance` 页面运行帅API模块。
- 从同源 `/api/perf-metrics/groups?hours=24|168` 读取性能数据。
- 分组倍率作为价格代理；没有请求数据的分组不参与推荐。
- 选择具体模型时，使用该模型在分组内的成功率、TTFT、延迟；“全部模型”使用分组汇总指标。
- 不读取、保存或展示完整 API Key、Cookie、Token；只读取密钥的 ID、名称和当前分组用于选择。
- 用户明确点击并确认后，调用帅API原生分组更新接口修改选中密钥的分组。

## 推荐规则

- “最低倍率”显示有数据且覆盖所选模型的最低倍率分组，即使该分组有警告也明确标注。
- “可靠推荐”默认要求成功率至少 99%，并排除描述中明确标记为不稳定/警告的分组。
- 可靠推荐排序为：倍率升序、成功率降序、TTFT 升序、平均延迟升序、名称稳定排序。
- 模型列表由接口实际返回的模型名动态生成，避免硬编码供应商类型。

## UI

- 右下角圆形“帅”按钮打开/隐藏悬浮卡片，支持最小化。
- 卡片包含时间窗（24h/7d）、模型选择、可靠性阈值、排除警告开关、刷新按钮、密钥和目标分组选择。
- 显示最低倍率、可靠推荐和前五个候选分组，展示倍率、成功率、TTFT、延迟、模型数量及警告。
- 每 60 秒刷新一次；接口失败保留上次数据并显示错误。

## 与现有代码的关系

AIHub 的请求、设置、切换和 UI 保持原行为；新增帅API常量、数据归一化/排序函数、密钥读取/分组更新适配和独立控制器，使用不同 DOM ID 和样式前缀避免冲突。脚本通过 hostname/path 路由启动对应控制器。
